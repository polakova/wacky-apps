# 🤪 111 Wacky Apps

111 tiny, useless, wonderful web toys — old-internet soul, modern polish.
Free to play in the browser; pay a coffee tip to take one home as a real app.

**Live:** http://138.68.81.78:8111/

---

## What's here

```
index.html              ← the hub (generated by build.py — do not hand-edit)
apps.json               ← the master list of all 111 apps (id, slug, title, emoji, category, blurb, spec)
build.py                ← generates index.html + apps/_shared/apps-list.json from apps.json
build_downloads.py      ← generates the take-home files: standalone HTML, bundle .zip, offline all-in-one site
STYLE_GUIDE.md          ← rules every app follows

apps/<slug>/index.html  ← one self-contained app per folder (vanilla HTML/CSS/JS, fully offline)
apps/_shared/wacky.css      ← shared neo-retro theme (every app links this)
apps/_shared/wacky-buy.js   ← the "keep this app" tip widget (floating button + Stripe modal), auto-injected
apps/_shared/apps-list.json ← generated app index for the buy widget's picker

thanks/index.html       ← post-payment page: Android → install .apk, iPhone → Add to Home Screen, desktop → web file

android/                ← Android WebView wrapper that turns any app into an .apk
.github/workflows/build-apks.yml ← CI that builds an .apk per app + one all-in-one, and publishes them
```

## How payments + downloads work

1. Every app page shows a **floating "💾 Keep this app"** button (injected by `apps/_shared/wacky-buy.js`).
2. The modal offers: this app (**€1**), any other app (**€1**), or **all 111** (**€5**). Choice is stored in `localStorage`.
3. It redirects to a **Stripe Payment Link**. Configure the two links at the top of `apps/_shared/wacky-buy.js` (`WACKY_PAY`).
4. **Both Payment Links must redirect after payment to** `http://138.68.81.78:8111/thanks/`
   (Stripe dashboard → each link → *After payment → Redirect* → that URL).
5. `thanks/index.html` reads the choice and serves the right download for the visitor's device:
   - **Android** → the installable **`.apk`** from this repo's `apks` Release
   - **iPhone/iPad** → the self-contained web file + "Add to Home Screen" steps (Apple doesn't allow .apk)
   - **Desktop** → both options

This is a **soft gate** by design: the apps are free and their source is public, so paying is a support tip, not DRM.

## Building the APKs

APKs are built in **GitHub Actions** (`.github/workflows/build-apks.yml`) — the Android SDK is too heavy for the web server.

The workflow:
1. Sets up JDK 17 + Android SDK 34 + Gradle 8.7 + Pillow + the Noto color-emoji font.
2. Runs `build.py` and `build_downloads.py` to produce the offline files.
3. For **each app**: drops its standalone HTML into `android/app/src/main/assets/www/`, generates an emoji launcher icon (`android/make_icon.py`), and runs `gradle assembleDebug` with a unique `applicationId` (`com.wackyapps.<slug>`) and the app's title as the label → `out/<slug>.apk`.
4. Builds the **all-in-one** app (the whole offline site, hub + 111 toys) → `out/all-111-wacky-apps.apk`.
5. Publishes every `.apk` to the **`apks`** GitHub Release (stable download URLs).

**Run it:** push to `main` (paths under `apps/`, `android/`, etc. trigger it), or *Actions → Build Wacky App APKs → Run workflow*.

Download URLs are then:
`https://github.com/polakova/wacky-apps/releases/download/apks/<slug>.apk`
`https://github.com/polakova/wacky-apps/releases/download/apks/all-111-wacky-apps.apk`

APKs are **debug-signed** so they sideload easily (Android asks to "allow from this source"). They request **no INTERNET permission** — every app is 100% local.

## Local development

```bash
python3 build.py            # rebuild hub + app list
python3 build_downloads.py  # rebuild take-home files
# serve locally:
python3 -m http.server 8111
```

## Deploying the site (tanyas)

The site is served as static files by Caddy on port 8111. Deploy with:

```bash
rsync -az --delete --exclude apps.json --exclude 'build*.py' --exclude STYLE_GUIDE.md \
  --exclude .git --exclude android --exclude .github --exclude README.md \
  ./ tanyas:/var/www/wacky-apps/
```

(Run `build.py` + `build_downloads.py` first so `index.html` and `download/` are current.)
